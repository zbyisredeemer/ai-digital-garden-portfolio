### 银行 eDDA 资金链路系统｜绑卡 / 入金 / 出金 / 对账补偿

**项目角色：核心后端开发 / 资金链路开发负责人之一**

**项目周期：2023.06 - 至今**

**技术栈：Java 8、Spring Boot、MySQL、RabbitMQ、XXL-JOB、JPA / MyBatis、HTTPS、Logback**

#### 项目简介

该项目负责公司与银行侧 eDDA 能力对接，支持用户在线完成银行卡绑定、入金、出金等资金操作。系统涉及 BOC、DBS 等银行通道，包含银行接口调用、同步响应处理、异步回调、状态流转、补偿重试、异常码处理和数据对账等核心能力。

资金链路对一致性、安全性、可追踪性要求较高，不能只依赖单次银行接口返回结果，需要通过同步响应、异步回调、定时补偿和日终对账共同保证最终状态一致。

#### 主要职责

- 参与 eDDA 绑卡、入金、出金业务流程设计，梳理银行请求、同步响应、异步回调、状态更新和异常补偿链路。
- 设计资金业务状态流转模型，处理申请中、处理中、成功、失败、待补偿、已补偿等状态。
- 参与 BOC、DBS 等银行通道接口对接，处理绑卡、入金、出金及银行回执数据。
- 开发银行异步回调处理逻辑，支持 FPS、GPC、ARE、ICC 等回执或对账数据处理。
- 设计定时补偿任务，对银行未回执、接口超时、网络异常、状态长时间处理中等场景进行扫描和补偿。
- 参与幂等控制设计，通过业务流水号、银行流水号、唯一索引和状态判断避免重复处理。
- 梳理银行异常码规则，区分网络异常、可补偿异常、业务失败异常和无需重试异常。
- 编写资金链路排查 SQL、异常处理文档和补偿操作说明，提升生产问题处理效率。

#### 关键难点与设计

- **状态机驱动资金流转**：将绑卡、入金、出金抽象为状态机模型，覆盖银行处理中、回调延迟、业务超时等复杂场景。
- **同步 + 异步 + 对账三层确认**：同步响应只作为初步结果，最终状态结合银行异步回调和对账数据确认。
- **补偿任务兜底**：通过定时任务扫描异常状态，对超时、未回执、处理中状态进行二次确认，保证最终一致性。
- **幂等保护机制**：通过业务流水号、银行流水号、状态判断和唯一约束处理重复请求、重复回调、重复补偿问题。
- **异常码分层处理**：将银行异常码按可重试、不可重试、业务失败、网络异常等类别处理，避免盲目重试带来资金风险。
- **对账兜底机制**：通过银行 ARE / ICC 等对账数据反查本地流水，识别本地遗漏、状态不一致或回调丢失的场景。

#### 项目价值

该系统支撑了用户在线绑卡、入金、出金等核心资金业务。通过状态机、异步回调、定时补偿、幂等控制和对账机制，提升了资金链路的稳定性、可追踪性和最终一致性，降低了人工处理和生产风险。

### 辉立暗盘行情接入与实时推送系统｜行情订阅 / WebSocket / 实时推送

**项目角色：核心后端开发 / 行情接入开发负责人之一**

**项目周期：2023.06 - 至今**

**技术栈：Java 8、Spring Boot、WebSocket、PMP SDK、定时任务、Logback、Linux**

#### 项目简介

公司需要接入辉立暗盘行情，为内部交易系统或 C++ 下游服务提供实时暗盘行情数据。项目通过 Java 服务接入辉立 PMP 行情源，完成行情连接、登录认证、股票订阅、行情接收、数据解析、快照维护、增量处理和 WebSocket 推送能力。

#### 主要职责

- 参与辉立 PMP 行情接入服务设计，梳理连接建立、登录认证、行情订阅、数据接收、断线重连、下游推送等核心流程。
- 基于 Java 8 和 Spring Boot 搭建独立行情接入服务，封装上游 PMP SDK 调用。
- 设计股票订阅清单同步机制，通过定时任务动态读取当天需要订阅的股票列表。
- 开发行情接收和解析逻辑，将上游 PMP 字段转换为内部标准行情模型。
- 维护股票行情快照，处理核心行情字段的增量变化。
- 设计 WebSocket 推送机制，将标准化行情数据推送给 C++ 或其他下游服务。
- 处理模拟环境和真实 PMP 环境差异，包括连接地址、账号密码、订阅字段和行情数据校验。
- 通过本地日志与线上真实行情数据对比，验证行情接收和字段解析准确性。

#### 关键难点与设计

- **行情接入服务独立化**：将行情连接、订阅、解析和推送逻辑独立成服务，降低对核心交易系统的侵入。
- **动态订阅机制**：通过定时任务持续同步当天股票清单，支持服务运行期间追加新增订阅，避免频繁重启。
- **快照 + 增量处理模型**：按股票维护本地行情快照，并根据上游增量字段更新内部行情状态，便于下游获取完整行情。
- **连接生命周期管理**：对登录、订阅、断线、重连、关闭等状态进行统一管理，提高行情服务稳定性。
- **WebSocket 实时推送**：通过长连接方式向下游服务推送行情数据，降低轮询开销，提升实时性。
- **日志校验机制**：记录关键行情字段和接收时间，通过对比线上真实行情验证本地接入链路准确性。

#### 项目价值

该项目完成了辉立暗盘行情从上游接入到下游推送的完整链路建设，为公司暗盘行情展示、交易辅助和数据验证提供了基础能力。通过动态订阅、快照维护、增量处理和日志校验机制，提高了行情服务的可维护性和可排查性。
